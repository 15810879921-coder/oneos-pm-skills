# 无单快轨到待开发

适用：口令明确「快轨」或「推进至待开发」且当前需求尚无标准分析路径任务（或用户确认跳过分析）。

## 动作表

| 对象 | 动作 |
|---|---|
| 需求 | 状态 → **待开发**（允许从待处理/已确认等快轨跳入；连跳按云效状态机） |
| 【交付】 | 无则新建；ASSOCIATED→需求（勿用 PARENT）；计划开始仅空时写当日；负责人→何斐；不写计划完成；描述先占位（同时设计完成+原型则走步骤4）；更新编号区块 |
| 【设计】 | **必须新建**；**TASK_SUB→交付**（`parent`+`parentIdentifier`+`createWorkitemRelationInfo=TASK_SUB`，交付「子项」须可见）；计划开始=当日（仅空时）；**同一轮收口**：计划完成=当日 + 阶段日历工时 + 任务完成态；描述一行：`快轨占位：未走完整设计，实质说明见交棒后补 AutoPRD`；更新编号区块 |
| 【分析】 | **禁止创建** |

## AutoPRD

快轨交棒**不**自动跑 AutoPRD（除非口令同时设计完成+原型）。  
交付仍占位 → Plan 勾选 §交棒占位风险，回报首行标红。

## 查重

只认任务编号。已有分析单不得因「快轨」被删除。

## 回报必含

- 需求 / 交付 / 设计编号
- 「快轨：未建分析；设计已占位收口」
- 若交付占位：红色风险提示

## 与标准 / 编号直推对比

```text
标准：分析中→交付+分析 → 设计中→设计+收口分析 → 设计完成→收口设计 → 待开发交棒
快轨（无既有阶段单）：→ 待开发：新建交付+设计+交棒何斐（无分析；设计当日收口）
编号直推：已有分析/设计编号 → 收口空计划完成 → 待开发+交付交棒（不新建冗余单）
```
