# AutoRDO 清洗与拆解细则

## 目标

碎片 → 可入库的书面「标题」与「描述」。保留原意；不写成产品 PRD。

## 必做与精炼规则

| 规则 | 说明 |
|---|---|
| **标题提炼** | 根据材料内容理解，自动概括简炼、清晰的标准需求标题：<br>• **内容理解**：准确识别归属的业务模块与核心动作/诉求（如：`验车管理：采购及租赁合同自动生成验车记录与入库控制` 或 `采购/租赁合同自动形成验车记录及入库控制`）。<br>• **自然流畅**：表达自然书面化，无需死板强求拼接固定句式；彻底剔除“我们要做一个”、“主要是用于”、“完成之后才能”等口语废话。<br>• **字数适中**：控制在 10-30 字内，能让读者一眼看懂需求核心。 |
| **描述转译** | 原文的转译结果，将口述/碎片材料转译为规范书面表达，保留原意（不改变谁要求什么、约束与例外；不替换成「更优方案」） |
| 书面化 | 口语改成完整句子或清晰短条目；主谓齐全 |
| 去口头禅 | 删除：嗯、啊、那个、就是说、然后呢、对对对、你懂吧 等 |
| 自我修正 | 以最后一次明确口径为准；若前后矛盾且未决，列入「待确认」 |
| 去重复 | 同一意思多轮确认只保留一处 |
| 轻度格式 | 可用短标题/条目；禁止展开成总览/角色/流程图/故事点等 PRD 结构 |
| 去结尾句号 | 描述中每个句子或条目末尾去掉 `。`；保留 `？` `！` 若确为疑问/感叹 |
| 待确认 | 材料含糊处写 `待确认：…`，不臆造 |

## 标题提炼对比示例

- ❌ **坏标题（直抄口语/带废话）**：`我们现在要做一个验车管理功能主要是采购合同和三方租赁合同自动形成验车记录验车完成之后才能车辆入库`
- ✅ **好标题（自然清晰，覆盖核心）**：`验车管理：采购与三方租赁合同自动生成验车记录及入库校验` 或 `合同签署后自动生成验车记录及车辆入库控制`

## 不做

- 不生成 AutoPRD 十章或「产品说明」
- 不写验收清单、故事点、mermaid
- 不改云效、不建【交付】/分析/设计
- 不加载 `yunxiao-requirement-lifecycle`

## 录音

1. 已有转写 → 按上文清洗转写稿并拆解标题与描述  
2. 仅音频 → 回报「请提供转写文本后再 AutoRDO」，停止声称已清洗完成  

## 质量自检

- [ ] **标题清晰通顺**（准确概括模块与核心动作，无死板句式绑定，无口语废话）  
- [ ] **描述为原文的转译结果**，干系人能认出这是自己说的意思  
- [ ] 无口头禅堆砌、无未决矛盾被写成定论  
- [ ] 描述末尾无 `。`  
- [ ] 未冒充完整 PRD
