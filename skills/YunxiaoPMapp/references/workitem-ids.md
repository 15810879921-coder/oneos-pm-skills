# 工作项编号（系统）

## 权威源

需求描述固定区块（机器可解析；人工只读即可）：

```markdown
## 工作项编号（系统）
- 交付：ONEOS-xx
- 分析：ONEOS-yy（无则写无）
- 设计：ONEOS-zz（无则写无）
```

## 规则

1. **禁止**仅靠本地会话映射当唯一真相。
2. 每次新建/登记交付·分析·设计编号后，**立即 PATCH 更新该区块**（只改本区块，不动 AutoRDO / AutoPRD 正文）。
3. 操作顺序：口令显式编号 > 读本区块 > ASSOCIATED/SUB 反查校验。
4. 三者冲突 → 停下人工；**仍禁止按标题猜**。
5. 本地缓存仅加速，启动以云效该区块为准。
6. 区块出现两个交付编号 → 停止并列号请人合并。
7. 取消需求：关联任务标取消/废止；**不删**本区块（留痕）。
